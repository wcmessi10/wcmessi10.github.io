1. In-memory Database 
    
    데이터를 메모리에 저장 및 관리하는 데이터베이스
    
2. In-memory Database 특징
    1. 메모리에 저장하므로 휘발성 데이터
3. Redis란
    1. 고성능 인메모리 데이터베이스
    2. NoSQL Database
4. Redis 특징
    1. 키 - 값 저장소 활용
    2. 다양한 자료 구조 지원 - ex) List, String, Set, Sorted Set, Hash
    3. TTL 지원 - 일정 시간 지나면 데이터 삭제 가능
    4. Single Thread - 각 명령들이 순서대로 진행되기 때문에 동시성 문제 해결
    5. 데이터 백업 지원 - 메모리는 휘발성 데이터 저장소지만, Redis에서 데이터를 안전한 저장소로 저장하는 기능 지원 ex) AOF, RDB
        1. AOF(Append Only File) - 수행된 명령어를 로그 파일에 기록, 데이터 복구를 위해 로그를 재실행
        2. RDB - 스냅샷 같은 기능, 저장된 데이터를 주기적으로 파일로 저장
5. Redis Cluster
    
    확장성, 부하 분산, 고가용성을 위한 분산 시스템 구성 방법 제공
    
    수평 확장이 가능하게 도와주는 기능
    
    샤딩 지원 - 논리적(slot), 물리적(shard) 샤드 지원
    
    서버 추가 시 - 자동으로 데이터 분산
    
    데이터 Replication 기능 지원 
    
6. Redis 활용 케이스
    1. 고성능 작업
    2. 캐시 - 매번 디스크에 접근하여 저장 및 조회하는 건 느리기 때문에, Redis에 일정 시간 캐시해둘 수 있다.
    3. Pub/Sub - Redis는 메시지를 발행 및 구독을 할 수 있어 실시간 통신에 활용 가능
7. Redis - Spring Application 활용
    1. 라이브러리 : 
        1. 동기 : org.springframework.boot:spring-boot-starter-data-redis
        2. 비동기 : org.springframework.boot:spring-boot-starter-data-redis-reactive
    2. yml → spring: data: redis: host 추가
    3. 활용 : RedisTemplate
    4. Lettus라는 라이브러리 통해 연결하는 방식도 있음 → 추후 자료 서치 예정
8. 분산 락 - 분산 시스템에서 락을 걸 수 있다. ex) Redis의 TTL을 활용한 락
